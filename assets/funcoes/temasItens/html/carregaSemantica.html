Semântica

Essa aqui é basicamente a razão da existência de HTML5. Até o HTML4 era comum que o próprio HTML tivesse tags que permitissem alterar os estilos dos elementos das páginas, porém isso foi caindo em desuso principalmente com a vinda do HTML5. Atualmente, essa função de estilização pertence ao CSS e, por mais que navegadores ainda possam aplicar essas tags HTML depreciadas (antigas, atrasadas), o correto é não utilizá-las, mas sim as CSS.

Certo, mas para que então serve HTML5? Como vimos anteriormente, HTML é uma linguagem de marcação de texto. Como os estilos dessa linguagem devem deixar de ser utilizados, então a função dela se mantém (firme e forte) a de marcação. E é aí que entra o título desse texto: a semântica.

Semântica é a mesma coisa que significado. Todas as tags HTML tem um significado, até mesmo uma genérica tipo <div></div> (que significa qualquer tipo de divisória). Por isso, todo mundo que desenvolve utilizando HTML deve ter uma atenção especial com o que digita, mesmo que aquilo não seja visto por quem usa o programa.

É bem fácil de compreender a semântica das tags já que elas geralmente tem nomes que deixam isso claro, como <img> para imagens, <p></p> para parágrafos, ou a já citada <div></div>. Porém existem outras, por exemplo, similares à <div></div>, que também não tem um efeito visual, mas que servem para deixar seu código mais profissional.

Um exemplo desse tipo de tag é a <section></section> que, repetindo, tem quase a mesma função que <div></div> (principalmente no visual), porém diz ao navegador que aquela é uma seção de conteúdo com mais significado. É usada simplesmente assim:

<!DOCTYPE html> <html lang="pt-br"> <head>   <meta charset="UTF-8">   <title>Document</title> </head> <body>   <section>
    <p>Aqui pode colocar qualquer tipo de conteúdo normalmente, o parágrado é só um exemplo</p>
  </section> </body> </html>

Seguindo a mesma linha de <section></section>, existe <aside></aside> para qualquer tipo de conteúdo "extra" do site, como aquelas partes menores nos cantos com menus e coisas do tipo ou até um trecho realmente extra depois do seu principal. Falando em principal, uma tag que também é similar, mas agora para definir o conteúdo principal, é <main></main>. Só para dar mais um exemplo, existe <footer></footer> para o rodapé da página, aquele lugar no final que geralmente tem "Quem somos", "SAC", um telefone ou qualquer outro tipo de informação que o proprietário ache útil.
